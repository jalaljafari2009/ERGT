# Theoretical Foundation

## 1. Foundational Thesis

ERGT begins from a relational thesis:

```text
An informational system is fundamentally a system of interactions.
```

Objects, vectors, concepts, memory, geometry, and reasoning are not treated as
primitive entities. They are treated as stabilized, compressed, or navigable
forms of relational structure.

The theory does not deny that vectors, tokens, or objects are useful. It denies
that they are the deepest explanatory level for intelligence. In ERGT, vectors
are computational coordinates. Relations are the primary structure to be
studied.

## 2. Relational Ontology

The basic entity of ERGT is not an object:

```text
Fundamental entity: interaction
Relational state: R(t)
```

`R(t)` denotes a time-dependent network of interactions. The system is not first
defined as points in a pre-existing space. Instead, space-like structure appears
when relations become stable enough, dense enough, and compressible enough to be
represented geometrically.

This is the central inversion:

```text
Classical view:     Space -> Distance -> Relation
ERGT view:          Relation -> Distance -> Geometry
```

In the classical view, objects exist in a space and distance determines how they
relate. In ERGT, relations exist first, and distance is an encoding of relational
strength.

## 3. Emergence Chain

ERGT organizes its theory around the following emergence chain:

```text
Dynamics -> Relations -> Structure -> Compression -> Geometry -> Memory -> Reasoning -> Intelligence
```

Each stage is a transformation in how interaction patterns become more stable,
more compact, and more usable.

## 4. Dynamics

Dynamics means that the system changes through interaction. The system is not a
static set of representations. It is a process in which representations affect
relations, relations affect geometry, and geometry affects subsequent
computation.

In implementation terms, hidden states change across layers and training steps.
In theoretical terms, these changes are not merely updates to vectors; they are
updates to the relational field generated by the system.

## 5. Relations

A relation is an interaction strength between parts of the system.

For the first operational version, relations are estimated from hidden-state
correlations:

```text
K_ij = sigmoid((x_i^T x_j) / sqrt(d))
```

This equation is not the full theory. It is the first computational probe of the
theory. The deeper claim is that meaningful informational structure should be
recoverable from patterns of interaction, not only from individual state vectors.

## 6. Structure

Not every relation matters equally. Structure emerges when relations become:

- Strong
- Stable
- Repeated
- Compressible
- Predictive of future computation

Structure is therefore not just a graph. It is a graph with persistence and
regularity.

In ERGT, a relational graph `W` becomes scientifically interesting only if it is
not arbitrary noise. It must show measurable organization such as clusters,
pathways, low-dimensional tendencies, semantic alignment, or layer-to-layer
persistence.

## 7. Compression

Direct relations can be expensive. A complete relation matrix stores pairwise
interaction strengths, but intelligence cannot depend only on enumerating all
pairwise facts.

Compression is the process by which recurring relational patterns become
represented in a lower-cost form.

The theoretical claim is:

```text
Geometry is a compressed representation of relations.
```

This is why geometry matters. Geometry is not added as decoration. It is the
form taken by relations when they become stable and compressible.

## 8. Distance

Distance is not primitive in ERGT. It is derived from relation strength.

The first operational distance is:

```text
D_ij = -log(W_ij + epsilon)
```

This transformation expresses a simple principle:

```text
stronger relation -> lower distance
weaker relation   -> higher distance
```

Distance becomes a cost of traversal through relational structure. It turns
interaction intensity into a geometry-like object that can guide attention.

## 9. Geometry

Geometry is the organized structure induced by distances.

In ERGT, geometry should not be understood merely as Euclidean coordinates.
Geometry means a relational distance structure that supports:

- Locality
- Neighborhoods
- Paths
- Clusters
- Barriers
- Stable regions
- Possible traversal

The project should therefore evaluate geometry through properties of `D`, not
only through visualizations. Useful geometry should show stability, non-trivial
organization, and explanatory power over model behavior.

## 10. Concept

A concept is not treated as an isolated symbolic object.

In ERGT:

```text
Concept = stable localized relational structure
```

A concept is a region in relational geometry that remains coherent across
contexts, perturbations, or layers. It is not merely a token, an embedding, or a
label. Those are coordinates or names for something deeper: a stable pattern of
relations.

This definition implies that concept formation should eventually be studied
through the persistence and localization of relational structures.

## 11. Memory

ERGT defines memory relationally:

```text
Memory = persistence of relational structure
```

Memory is not only stored symbols, cached activations, or external retrieval. A
system remembers when a relational structure persists strongly enough to affect
future computation.

This makes graph memory a natural extension of the first proof stage:

```text
W(l+1) = (1 - eta)W(l) + etaK
```

However, memory is not part of the first required proof. It becomes justified
only if the project first shows that relational distance is useful for
attention.

## 12. Reasoning

Reasoning is defined as traversal through relational geometry:

```text
Reasoning = navigation of stable relational structure
```

This does not reduce reasoning to shortest paths. A path is only one possible
operational form. The deeper point is that reasoning requires moving through a
structured space of relations, constraints, and stable regions.

If concepts are stable relational regions, reasoning is the controlled movement
between such regions.

This claim belongs to the long-term theory. It should not be asserted as proven
by the first attention experiment.

## 13. Intelligence

The broad ERGT definition of intelligence is:

```text
Intelligence = discovery, compression, stabilization, and traversal of
relational structures
```

This definition combines four requirements:

- Discovery: finding relations that matter.
- Compression: reducing relational complexity into usable structure.
- Stabilization: preserving useful structure across time and context.
- Traversal: using structure to reason, infer, or act.

The first ERGT proof stage tests only the beginning of this definition. It asks
whether discovered relations can induce distances that improve attention.

## 14. Relation to Transformers

Transformers already contain a relational mechanism: attention.

Standard attention computes interaction through dot products:

```text
A = Softmax(QK^T / sqrt(d))
```

ERGT does not reject attention. It reinterprets and extends it.

The standard transformer computes attention from immediate vector similarity.
ERGT asks whether attention should also be shaped by an emergent relational
geometry:

```text
A = Softmax(QK^T / sqrt(d) - alphaD)
```

Here `D` is not an external position bias. It is derived from the model's own
relational structure. This is the architectural bridge between the theory and
the experiment.

## 15. Coordinate Substrate vs Relational Reality

ERGT still uses hidden vectors `X`:

```text
S = (X, W)
```

This does not contradict the relational thesis.

`X` is the coordinate substrate used by the machine. `W` is the relational
structure extracted from that substrate. The theory gives priority to structures
that remain meaningful beyond the raw coordinate representation.

This distinction matters because otherwise ERGT would collapse into ordinary
embedding analysis. The project should seek relational invariants, not merely
larger or different vectors.

## 16. The First Scientific Reduction

The full theory is broad:

```text
Relations -> Structure -> Compression -> Geometry -> Memory -> Reasoning -> Intelligence
```

The first experimental reduction is narrow:

```text
Correlation -> Distance -> Geometry -> Attention
```

This reduction is not a betrayal of the theory. It is the first controlled test
of the theory.

The project must protect both levels:

- The broad theory gives meaning and direction.
- The narrow experiment gives falsifiability and scientific discipline.

### Post-Phase-3 Strengthening

The early Phase 3 evidence showed that the first reduction is necessary but not
sufficient. A useful result against baseline is not enough if random or shuffled
distance can explain the same gain. ERGT therefore strengthens the operational
reduction for post-Phase-3 work:

```text
HiddenStates
-> W relations
-> Phi information potential
-> W_t stable memory
-> causal shortest-path geometry
-> validated GeoAttention
-> reasoning paths
```

This strengthened path is defined in
`docs/17_physics_aligned_ergt_program.md`. It preserves the relational thesis
while adding explicit tests for control fairness, reconstructibility, causal
reachability, stability, and anti-collapse behavior.

## 17. Falsifiability

ERGT must remain testable.

The first mechanism is weakened if:

- Real relational distance behaves like random or shuffled distance.
- The distance term duplicates ordinary attention without adding useful signal.
- Relational graphs collapse into trivial uniform or identity-like matrices.
- Geometry metrics show no stability or organization.
- Improvements disappear across seeds or datasets.

If these happen, the current operational definition is wrong or incomplete. The
broader relational philosophy may still be valuable, but this implementation
path must be revised.

## 18. Guiding Principle

The guiding principle for the project is:

```text
Do not add geometry to the model.
Let geometry emerge from relations, then test whether that geometry changes
computation in a useful way.
```

This is what separates ERGT from ordinary attention bias, positional encoding,
or graph augmentation. The geometry must be induced, measured, and tested as a
consequence of relational dynamics.
